Typische Beispiele sind Zeit sowie motorische und mentale Anstrengung. --------- Review: Evaluierung ========= ========= Eine von Laien, d.h. von Menschen mit begrenzter Usabilityerfahrung, durchgeführte Inspektion. --------- Rolle: Anforderungen, Rollen ========= Role ========= Ein Menge zusammenhängender Aspekte wie Verhalten, Rechte, Verpflichtungen und Normen in einer Arbeitssituation. Abhängig von der Komplexität eines Projektes können mehrere Menschen eine Rolle teilen oder mehrere Rollen einer Person zugewiesen werden. --------- Sekundäre Benutzer: Rollen, Planung ========= secondary user ========= Menschen, die keine Benutzer sind, aber auf gewisse Weise vom interaktiven System betroffen oder beinflusst sind. Sekundäre Benutzer sind u.a. Führungskräfte von Benutzern, Vertriebspersonal und Vertreter, Projektmanager, Anwendungsentwickler, Trainer und das Kundenbetreuungsteam. Ist das interaktive System für die Benutzung zuhause gedacht, können sekundäre Benutzer auch Freunde oder Verwandte und Besucher des Benutzers sein. --------- Selbstbeschreibungsfähigkeit: Gestaltung, Prinzipien ========= Usability Affordance Self-discriptiveness ========= Ein Dialogprinzip. Ein Dialog ist xxxx in dem Ausmaß, in dem es zu jeder Zeit dem Benutzer offensichtlich ist, in welchem Dialog er sich gerade wo befindet und welche Aktionen wie durchgeführt werden können. --------- Steuerbarkeit: Gestaltung, Prinzipien ========= Controllability ========= Ein Dialogprinzip. Ein Dialog ist xxxx, wenn der Benutzer in der Lage ist, den Dialogablauf zu starten sowie seine Richtung und Geschwindigkeit zu beeinflussen, bis das Ziel erreicht ist. --------- Storyboard: Planung, ========= ========= Eine Folge visueller Bildschirminhalte, die das Zusammenspiel zwischen einem Benutzer und einem vorgestellten interaktiven System darstellt. Ein Storyboard ähnelt oft einem Zeichtrickfilm. --------- Styleguide: Gestaltung, Prinzipien, ========= Gstaltungsregel UX Guides Richtlinie ========= Eine Sammlung von Dialogprinzipien und Gestaltungsregeln die verwendet wird, um die grundsätzliche Erscheinung der Benutzungsschnittstelle auf jeglicher Software einer Organisation festzulegen. --------- Suche: Prinzipien, System ========= Quest Search ========= Eine Komponente der Benutzungsschnittstelle, die zum (Wieder)Auffinden von Nutzungsobjekten auf Basis fragmentaler Informationen zu diesem Objekt herangezogen wird. --------- Suggestivfrage: Evaluierung, Test ========= Leading question ========= Eine Frage in einem Interview, die Annahmen oder einen Rahmen voraussetzt, der eine Präferenz für bestimmte Antwortmöglichkeiten vorgibt oder versucht, die Antwort in eine bestimmte Richtung zu lenken. --------- Summative Evaluierung: Evaluierung, ========= Summative usability evaluation ========= Eine Usabilityevaluierung zur Feststellung, ob ein interaktives System die Nutzungsanforderungen erfüllt. Summative Evaluierungen werden auch benutzt, um den Grad der erreichten Gebrauchstauglichkeit eines interaktiven Systems zu bestimmen. Summative Evaluierungen finden in der Regel dann statt, wenn ein interaktives System oder wesentliche Teile davon fertig entwickelt sind. --------- Testaufgabe: Planung, Test, ========= Usability-Testaufgabe Usability test task ========= Eine Beschreibung einer typischen Aufgabe, die ein Moderator einem Testteilnehmer während eines Usabilitytests stellt. Manchmal als Testszenario bezeichnet. --------- Testskript: Test, Evaluierung ========= Usability Checkliste Test script Usability ========= Eine von einem Moderator in einem Usabilitytest verwendete Checkliste, um den Überblick über die Fragen des Pre-Test-Interviews, die Testaufgaben und die Fragen des Post-Test-Interviews zu behalten. --------- Testteilnehmer: Rollen, ========= Benutzer Test participant ========= Repräsentativer Benutzer, der typische Aufgaben in einem Usabilitytest löst. --------- Umgebung: Planung, System, ========= Context Environment Nutzungskontext ========= Die Gesamtheit der gegebenen Umstände oder Bedingungen, in denen ein Benutzer arbeitet. Diese umfassen die physische und die soziale Umgebung. --------- Unbeaufsichtigter Usabilitytest: Test, ========= ========= Ein Usabilitytest bei dem Testteilnehmer Testaufgaben lösen ohne dabei beobachtet zu werden. Handlungen werden auf Video zur späteren Analyse aufgezeichnet. --------- Usability: Test, Anforderungen, Prinzipien ========= Gebrauchstauglichkeit ========= Ausmaß, in dem ein interaktives System oder ein Dienst durch bestimmte Benutzer in einem bestimmten Nutzungskontext genutzt werden kann, um festgelegte Ziele effektiv, effizient und zufriedenstellend zu erreichen. --------- Usability Engineer: Rollen ========= ========= Eine Rolle im Prozess der menschzentrierten Gestaltung. Der Usability Engineer betreut in einer Querschnittsfunktion verantwortlich den menschzentrierten Gestaltungsprozess. Er ist kompetent und erfahren in den Prozessschritten, verfügbaren Methoden der menschzentrierten Gestaltung und Gestaltungsregel für Benutzererlebnis, sodass er die Entwicklungsprojekte unter dem Aspekt menschzentrierter Gestaltung planen und die Qualität der Ausführung von menschzentrierter Gestaltungsbezogenen Aktivitäten sicherstellen kann. Dies beinhaltet die Integration der menschzentrierten Gestaltung in den Produktentwicklungsprozess des Unternehmens, die Definition von Erfolgskriterien für solche Projekte sowie das Training von beteiligten Projektteams. Er ist in der Lage, die für das jeweilige Projekt relevanten Arbeitsergebnisse, hierfür geeignete Vorgehensweisen sowie Arbeitshilfen und Festlegungen (z.B. Dialogprinzipien und Gestaltungsregel) für den menschzentrierten Gestaltungsprozess zu identifizieren und für ein Projekt festzulegen. --------- Usability Professional: Rollen, ========= ========= Eine Person, die qualifiziert und methodisch die Anforderungen an die Usability (Gebrauchstauglichkeit) interaktiver Systeme (Hardware und Software) herleitet, umsetzt oder deren Umsetzung überprüft. Der Usability Professional ist typischerweise in einem oder mehrerer der folgenden Arbeitsschwerpunkte spezialisiert: Analyse, Gestaltung, Evaluierung, Prozessgestaltung und Methodeneinsatz. Die Grundlage der Tätigkeiten eines Usability Professionals sind die internationalen Normen zur Usability interaktiver Systeme und deren Gestaltungsprozess (ISO 9241) sowie aktuelles, publiziertes Fachwissen. Usability Professionals arbeiten in Projekten zu Herstellung von interaktiven Systemen und mit ihnen verbundener Dienstleistungen und Prozesse für den Einsatz in zahlreichen Projektkontexten (z.B. interaktive Systeme für den gewerblichen Einsatz, Haushaltsgeräte, interaktive Spiele, mobile Endgeräte). --------- Usability Tester: Rollen, ========= Benutzer ========= Eine Rolle im Prozess der menschzentrierten Gestaltung. Der Usability Tester evaluiert Benutzungsschnittstellen in verschiedenen Stadien der Realisierung. Er definiert in Abstimmung der Prozessbeteiligten den Aufbau der Evaluierungen und verantwortet die valide Ausführung der Evaluierungen. Der Usability Tester erstellt entwicklungsbegleitende Usabilitytestberichte mit dem Ziel der Produktverbesserung sowie den Usabilitytestbericht über die abschließende summative Evaluierung. Er kommuniziert diese Ergebnisse an die Interessenvertreter (Projektbeteiligten) oder betreut deren Kommunikation verantwortlich. Dies kann bspw. im Rahmen eines Abnahmetests in Zusammenarbeit mit dem User Requirements Engineer erfolgen. --------- Usabilitylabor: System, Planung, ========= ========= Zwei oder mehrere Räume, die speziell für die Durchführung von Usabiltytests oder Fokusgruppen ausgestattet sind. Ein Usabilitylabor besteht of aus einem Testraum, wo die Testteilnehmer sitzen, und einem Beobachtungsraum, wo Interessenvertreter die Testteilnehmer beobachten können, während die Testteilnehmer Aufgaben lösen. Oft sind diese zwei Zimmer durch eine Spiegelwand getrennt, was den Beobachtern erlaubt, den Testteilnehmern zuzuschauen, nicht aber umgekehrt. --------- Usabilitytest: Test, Evaluierung ========= ========= Der Validierungsprozess zur Feststellung dass ein interaktives System alle vorher festgelegten Nutzungsanforderungen erfüllt. Ein Usabilitytest kann die Umsetzung der vorliegenden quantitativen und qualitativen Nutzungsanforderungen einschließlich Aufgabenvervollständigung, Aufgabendauer, Fehlerraten und Benutzerzufriedenstellung validieren. Ein Usabilitytest umfasst eine Reihe von Usabilitytestsitzungen. In jeder Sitzung versucht ein Testteilnehmer unter Benutzung des interaktiven Systems oder eines Prototypen eines interaktiven Systems repräsentative Aufgaben zu lösen. Üblicherweise werden Testsitzungen von einem Moderator moderiert und von einer Anzahl von Interessenvertretern beobachtet. Ein Protokollant zeichnet wichtige Befunde auf. Testen kann formal oder informal sein. Das Testen kann face-to-face oder remote stattfinden und qualitative oder quantitative Daten ergeben. Das Testen kann zu jedem Zeitpunkt während der Entwicklungsphase geschehen: ­ von der frühen Analysephase bis hin zur Lieferung des interaktiven Systems und darüberhinaus. Die Tests können sowohl an Papierentwürfen oder primitive Attrappen als auch an in Entwicklung befindlichen oder bereits fertiggestellten interaktiven Systemen durchgeführt werden. --------- Usabilitytestbericht: Test, Evaluierung ========= Testbericht Evaluierungsbericht Test report ========= Ein Dokument, das die Ergebnisse eines Usabilitytests beschreibt. Ein Usabilitytestbericht beinhaltet eine Kurzdarstellung, 5-50 Befunde und das Testskript für den Usabilitytest. Oft umfasst der Usabilitytestbericht auch relevante Screenshots oder Bilder des getesteten interaktiven Systems. Manchmal auch als Testbericht bezeichnet. --------- Usabilitytestplan: Planung, Evaluierung, ========= ========= Eine kurze Beschreibung des Zwecks und Umfangs eines Usabilitytests. Der Plan beinhaltet die Anzahl geplanter Testteilnehmer, die ungefähre Länge jeder Testsitzung, den Namen des Moderators, einen Zeitplan und eine Kostenschätzung für den Usabilitytest einschließlich Personenstunden. --------- Usabilitytestsitzung: Gestaltung, Test, ========= ========= Der Teil eines Usabilitytests, bei dem ein bestimmter Testteilnehmer repräsentative Aufgaben unter Benutzung des interaktiven Systems oder des Prototypen eines interaktiven Systems löst. --------- User Interface Designer: Rollen ========= Usability Professional ========= Eine Rolle im Prozess des menschzentrierten Designs. Ein User Interface Designer implementiert das Benutzererlebnis basierend auf dem Design und den Nutzungsszenarios, des Interaktionsdesigners. Der User Interface Designer erzeugt interaktive Prototypen. --------- User Requirements Engineer: Rollen, ========= ========= Eine Rolle im Prozess des menschzentrierten Designs. Der User Requirements Engineer identifiziert und beschreibt den tatsächlichen Nutzungskontext von Benutzern. Daraus leitet der User Requirements Engineer die Nutzungsanforderungen und damit verbundenen organisatorischen Anforderungen ab, die für ein bestimmtes Projekt umgesetzt werden müssen. Der User Requirements Engineer priorisiert die Anforderungen um sicherzustellen, dass die grundsätzlichen Erfordernisse des Benutzers erfüllt werden. --------- Wireframe: Planung, ========= Mockup Modell Attrappe ========= Grobe Skizze von Navigations- und Inhaltselementen, die eine Benutzungsschnittstelle ausmachen. Typischerweise werden dabei das visuelle Design und das exakte Layout ignoriert.
