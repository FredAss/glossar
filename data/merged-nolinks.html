Anforderungen:
  • Interessenvertreteranforderung
  • Marktanforderung
  • Organisatorische Anforderung
  • Nutzungsanforderung
--------- Arbeitsmittel ========= ========= Hardware, Software und Materialien. --------- Aufgabe ========= ========= Aktivitäten die erforderlich sind, um ein Ziel zu erreichen. --------- Aufgabenangemessenheit ========= ========= Ein Dialogprinzip. Ein interaktives System ist aufgabenangemessen, wenn es den Benutzer auf den charakteristischen Eigenschaften der Aufgabe basieren, anstatt auf der zur Aufgabenerledigung eingesetzten Technologie. Empfehlungen zu Befolgung des Dialogprinzips:
  1. Der Dialog sollte dem Benutzer solche Informationen anzeigen, die im Zusammenhang mit der erfolgreichen Erledigung der Aufgabe stehen.
  2. Der Dialog sollte dem Benutzer keine Informationen anzeigen, die nicht für die erfolgreiche Erledigung relevanter Aufgaben benötigt werden.
  3. Die Form der Eingabe und Ausgabe sollte der Aufgabe angepasst sein. Wenn für eine Aufgabe ganz bestimmte Eingabewerte typisch sind, sollten diese Werte dem Benutzer automatisch als voreingestellte Werte verfügbar sein.
  4. Die vom interaktiven System verlangten Dialogschritte sollten zum Arbeitsablauf passen, d. h., notwendige Dialogschritte sollten enthalten sein und unnötige Dialogschritte sollten vermieden werden.
--------- Aufgabenmodell ========= ========= Eine Beschreibung der Aktivitäten innerhalb einer Aufgabe die erledigt werden müssen, um die vom Benutzer beabsichtigen Ziele zu erreichen. --------- Barrierefreiheit ========= ========= Die Attribute und Charakteristiken eines interaktiven Systems die Menschen mit eingeschränkter Sicht, eingeschränktem Hören, eingeschränkter Geschicklichkeit, eingeschränkter Kognition oder eingeschränkter physikalischer Mobilität erlauben effektiv und effizient mit dem interaktiven System zu interagieren. --------- Befund ========= ========= Ergebnis aus einem Usabilitytest. Ein Befund kann ein Nutzungsproblem oder etwas sein, das die Benutzer mochten, also ein positiver Befund. --------- Benutzer ========= ========= Person, die mit einem interaktiven System arbeitet. --------- Benutzerdokumentation ========= ========= Eine Form der Benutzerunterstützung. Für Benutzer geschriebene Anleitungen darüber, wie das interaktive System durch Benutzer eingesetzt werden kann. --------- Benutzererlebnis ========= ========= Wahrnehmungen und Reaktionen einer Person, die aus der tatsächlichen und/oder erwarteten Benutzung eines interaktiven Systems oder Dienstes resultieren. --------- Benutzerfragebogen ========= ========= Ein Satz von Fragen, der eine strukturierte Art und Weise der Befragung einer großen Gruppe von Menschen ermöglicht, um diese Menschen, ihre Interessen und ihre Präferenzen zu beschreiben. Benutzerfragebögen können im Rahmen einer Benutzerbefragung benutzt werden, um die Zufriedenstellung von Benutzern eines interaktiven Systems und Informationen zum Nutzungskontext zu ermitteln. --------- Benutzergruppe ========= ========= Eine Gruppe von Benutzern mit demselben oder ähnlichen Nutzungskontext bezogen auf das interaktive System. --------- Benutzergruppenprofil ========= ========= Eine verallgemeinerte Beschreibung einer Benutzergruppe. --------- Benutzerunterstützung ========= ========= Ein gemeinsamer Ausdruck für Benutzerdokumentation, Onlinehilfe und Fehlermanagement. --------- Usabilitytest ========= ========= Evaluation, die repräsentative Benutzer involviert, die bestimmte Aufgaben mit dem interaktiven System ausführen, um die Identifikation von Nutzungsproblemen und/oder die Messung von Effizienz, Effektivität und/oder Zufriedenstellung zu ermöglichen. Siehe Usabilitytest. --------- Benutzungsschnittstelle ========= ========= Alle Bestandteile eines interaktiven Systems, die Informationen und Steuerelemente zur Verfügung stellen, die für den Benutzer notwendig sind, um eine bestimmte Aufgabe mit dem interaktiven System zu erledigen. --------- Benutzungsschnittstellenstandard ========= ========= Eine Sammlung von Richtlinien für die Gestaltung von Benutzungsschnittstellen. --------- Beobachter ========= ========= Eine Rolle in einem Usabilitytest. Beobachter sind Interessenvertreter, die schauen, was während eines Usabilitytests geschieht. Durch das Beobachten einer Usabilitytestsitzung erlangen die Beobachter Eindrücke von den Nutzungsproblemen und Erfolgen der Testteilnehmer aus erster Hand. --------- Beobachtung ========= ========= Eine Technik für das Sammeln von Kontextinformationen bezüglich Erfordernissen. Während einer Beobachtung wird dem Benutzer bei der Ausführung von Aufgaben am interaktiven System zugesehen. Der Beobachtende greift nicht ein, außer es besteht gelegentlich die Notwendigkeit zu einer klärenden Frage. --------- Designmuster ========= ========= Eine wiederverwendbare Lösung für ein Designproblem innerhalb eines Kontexts. Ein Designmuster beschreibt ein Designproblem, eine Lösung und wo sich diese Lösung als funktionierend herausgestellt hat. --------- Dialog ========= ========= Interaktion zwischen einem Benutzer und einem interaktiven System in Form einer Folge von Handlungen des Benutzers (Eingaben) und Antworten des interaktiven Systems (Ausgaben), um ein Ziel zu erreichen. --------- Dialogprinzip ========= ========= Allgemeines Ziel für die Gestaltung von Dialogen. Dialogprinzipien sind aus der Erfahrung abgeleitete, abstrakte Richtlinien für das Dialogdesign auf oberster Ebene. Aufgrund ihrer Allgemeingültigkeit kann es schwierig sein, sie im konkreten Kontext umzusetzen. Die DIN EN ISO 9241-110 führt folgende sieben Dialogprinzipien auf:
  1. Aufgabenangemessenheit;
  2. Selbstbeschreibungsfähigkeit;
  3. Erwartungskonformität;
  4. Lernförderlichkeit;
  5. Steuerbarkeit;
  6. Fehlertoleranz;
  7. Individualisierbarkeit.
--------- Dringlichkeitsstufen (von Befunden) ========= ========= Ein Maß für ein Nutzungsproblem aus einem Usabilitytest, das die Auswirkung des Nutzungsproblems auf das Benutzererlebnis bewertet. Der Usabilitytester bewertet die Nutzungsprobleme aus Sicht der Testteilnehmer. Typische Dringlichkeitsstufen sind: niedrig, ernst, kritisch, lebensbedrohend. --------- Effektivität ========= ========= Genauigkeit und Vollständigkeit mit der Benutzer ein bestimmtes Ziel erreichen. --------- Effizienz ========= ========= Im Verhältnis zur Genauigkeit und Vollständigkeit eingesetzte Ressourcen, mit dem Benutzer ein bestimmtes Ziel erreichen. --------- Erfordernis ========= ========= Eine für einen Benutzer oder eine Benutzergruppe als notwendig identifizierte Voraussetzung, um ein Ziel innerhalb eines bestimmten Nutzungskontextes zu erreichen. Beispiele:
  • Ein Vortragender (Benutzer) muss wissen, wieviel Zeit er noch hat (Voraussetzung), um den Vortrag während einer Präsentation mit einer festgelegten Zeitbeschränkung (Nutzungskontext) abschliessen zu können (Ziel).
  • Ein Finanzbuchhalter (Benutzer) muss die Anzahl der erhaltenen Rechnungen und ihre Beträge kennen (Voraussetzung) um die tägliche Rechnungsstellung (Ziel) als Teil der Cashflow-Kontrolle (Nutzungskontext) fertigstellen zu können.
--------- Erwartungskonformität ========= ========= Ein Dialogprinzip. Ein Dialog ist erwartungskonform, d.h. im Einklang mit den Benutzererwartungen, wenn er den aus dem Nutzungskontext heraus vorhersehbaren Benutzerbelangen sowie allgemein anerkannten Konventionen entspricht. Konsistenz ist ein Aspekt der Erwartungskonformität. --------- Evaluation ========= ========= Ein Prozess der Informationen über die Gebrauchstauglichkeit eines interaktiven Systems sammelt um das interaktive System zu verbessern (formative Evaluation) oder um eine fertiggestellte Benutzungsschnittstelle zu bewerten (summative Evaluation). Evaluation ist ein gemeinsamer Ausdruck für inspektionsbasierte und benutzerzentrierte Evaluation. --------- Expertenreview ========= ========= Eine Inspektion die von Usabilityexperten oft in Kombination mit Fachexperten durchgeführt wird. Ein Expertenreview basiert nur auf dem Wissen und den Erfahrungen des Experten. Heuristiken werden dabei nicht verwendet. --------- Fehlermanagement ========= ========= Ein Teil der Benutzerunterstützung. Merkmale der Benutzungsschnittstelle die Benutzern helfen, Fehler zu erkennen und den Einfluß von Fehlern zu minimieren. Bemerkung zum Seminar: Vergleiche mit Fehlertoleranz. --------- Fehlertoleranz ========= ========= Ein Dialogprinzip. Ein Dialog ist fehlertolerant, wenn das beabsichtigte Arbeitsergebnis trotz erkennbar fehlerhafter Eingaben entweder mit keinem oder mit minimalem Korrekturaufwand seitens des Benutzers erreicht werden kann.
  1. Wenn ein Fehler auftritt, sollte das interaktive System eine konstruktive, genaue und verständliche Erklärung anbieten.
  2. Wenn aus einer Benutzeraktion schwerwiegende Folgen entstehen können, dann sollte das interaktive System eine Erklärung anbieten und eine Bestätigung vor der Durchführung der Aktion geben.
--------- Fokusgruppe ========= ========= Eine gezielte Diskussion, bei der ein Moderator eine Gruppe von Teilnehmern durch eine Menge von Fragen zu einem bestimmten Thema führt. --------- Formative Evaluation ========= ========= Eine Art der Usability-Evaluation die hilft, das Design für ein interaktives System oder eine Dienstleistung zu formen. Formative Evaluationen beziehen die Evaluation eines interaktiven Systems oder einer Dienstleistung oft iterativ während der Entwicklung mit dem Ziel mit ein, Nutzungsprobleme frühzeitig zu erkennen und zu eliminieren. --------- Fragebogen ========= ========= Ein Satz von Testfragen zum Benutzererlebnis. Ein Fragebogen ist das tragende Instrument zur Messung der Zufriedenstellung eines Benutzers mit einem interaktiven System bei allen Arten der Evaluation einer Benutzungsschnittstelle. --------- Gebrauchstauglichkeit ========= Usability ========= Siehe Usability. --------- Geschäftsanforderung ========= Organisatorische Anforderung ========= Siehe Organisatorische Anforderung. --------- Geschlossene Frage ========= ========= Eine Interviewfrage die eine Antwort aus einem vordefinierten Satz von Alternativen fordert, oft einfach ,,ja" oder ,,nein". --------- Gestaltungsregel ========= ========= Konkrete, spezifische Instruktionen, denen ein Designer mit minimalem Interpretationsaufwand folgen kann. Gestaltungsregel ist ein allgemeiner Ausdruck für die Inhalte folgender Dokumententypen:
  1. Gestaltungsrichtlinie / Benutzungsschnittstellenrichtlinie
  2. Styleguide
  3. Standard / Benutzungsschnittstellenstandard
  4. Designmuster
--------- Handlungsleitung ========= ========= Aspekte eines Objektes, die klar machen, wie das Objekt benutzt werden kann. Beispiele:
  1. Ein Henkel einer Teetasse stellt eine offensichtliche Handlungsleitung dar, sie zu halten.
  2. Ein Button auf einer Webseite bietet die Handlungsleitung, auf ihn zu klicken.
  3. Ein Firmenemblem in der oberen linken Ecke einer Webseite bietet eingeschränkte Handlungsleitung, darauf zu klicken.
--------- Heuristik ========= ========= Ein allgemein anerkanntes Dialogprinzip zur Evaluation einer Benutzungsschnittstelle, das benutzt werden kann, um Designprobleme zu erkennen oder zu verhindern. --------- Heuristische Evaluation ========= ========= Eine Methode der Usability-Evaluation bei der ein oder mehrere Reviewer ­ vorzugsweise Experten ­ ein interaktives System mit einer Liste von Dialogprinzipien (auch als Heuristiken bezeichnet) vergleichen und feststellen, wo das interaktive System diesen Dialogprinzipien nicht folgt. --------- High-Fidelity Prototyp ========= ========= Ein Prototyp, der mit Softwarepaketen wie z.B. Visual Basic oder PowerPoint erstellt wurde. Ein High-Fidelity Prototyp ähnelt dem fertigen interaktiven System. --------- Indiviualisierbarkeit ========= ========= Ein Dialogprinzip. Ein Dialog ist individualisierbar wenn Benutzer die Mensch-System-Interaktion und Darstellung von Informationen an ihre individuellen Fähigkeiten und Bedürfnisse anpassen können. --------- Informationsarchitekt ========= ========= Eine Rolle im Prozess des menschzentrierten Designs. Der Informationsarchitekt kreiert und organisiert die Struktur für das effiziente Auffinden von Information in interaktiven Systemen durch den Nutzer. Er definiert hierbei die nutzergruppengerechte Aufbereitung von Inhalten und Navigationsstrukturen. Er ist für die konsistente und erwartungskonforme Bezeichnung von Wegweisern (z.B. in Menüs) sowie erforderlicher Nutzungsobjekte und Werkzeuge für den Nutzer zuständig. --------- Informationsarchitektur ========= ========= Die Klassifikation und Organisation von Information in einem interaktiven System. Bemerkung zum Seminar: Die Informationsarchitektur ist für Benutzer nicht sichtbar. Sie dient hauptsächlich als Basis der Navigationsstruktur für das Entwicklungsteam. --------- Inspektionsbasierte Evaluation ========= ========= Evaluation die auf der Beurteilung ein oder mehrerer Evaluatoren basiert, die ein interaktives System prüfen oder benutzen, um potentielle Nutzungsprobleme und/oder Abweichungen von anerkannten Kriterien zu identifizieren. Inspektionsbasierte Evaluation wird von Usability Experten oder Fachexperten durchgeführt, die ihre Urteile auf die frühere Erfahrung mit Nutzungsproblemen und ihr eigenes Wissen über Richtlinien und Standards zu Usability gründen. Inspektionsbasierte Evaluation ist ein Überbegriff für Expertenreview und heuristische Evaluation. --------- Interaktionsdesigner ========= ========= Eine Rolle im Prozess des menschzentrierten Designs. Der Interaktionsdesigner konzipiert und definiert die Interaktion zwischen Mensch und System auf Basis der Nutzungsanforderungen und Aufgaben der Nutzer, mit dem Ziel der effizienten Aufgabenerledigung am System. Er generiert hierzu Nutzungsszenarien, die die Effektivität, Effizienz und Zufriedenstellung bei der Aufgabenerledigung sicherstellen. Er berücksichtigt hierbei die Nutzungsanforderungen, identifiziert und klärt offene Fragen, die sich aus der spezifizierten Interaktion ergeben. --------- Interaktives System ========= ========= Kombination von Hardware- und Softwarekomponenten, die Eingaben von einem Benutzer empfangen und Ausgaben zu einem Benutzer übermitteln, um ihn bei der Ausführung einer Aufgabe zu unterstützen. --------- Interessenvertreter ========= Stakeholder ========= Einzelperson oder Organisation, die ein Anrecht, einen Anteil, einen Anspruch oder ein Interesse auf ein interaktives System oder an dessen Merkmalen hat, die ihren Erfordernissen und Erwartungen entsprechen. --------- Interessenvertreteranforderung ========= ========= Was das interaktive System aus Sicht des Interessenvertreters leisten muss. --------- Interviewcheckliste ========= ========= Eine schriftliche Liste passender Fragen und Hinweise, die während eines Interviews verwendet wird, um sicherzustellen, dass alle relevanten Themen berücksichtigt werden. --------- Intuitiv ========= ========= Die Benutzung des interaktiven Systems ist unmittelbar zu verstehen ­ unabhängig von der Erfahrung, des Wissens, der Sprachkenntnisse oder des momentanen Konzentrationsgrades des Benutzers. --------- ISO 9241 ========= ========= Eine Familie von Standards bestehend aus zwei Hauptgruppen. Gruppe 1 besteht aus Standards unter dem allgemeinen Titel "Ergonomische Anforderungen für die Büroarbeit mit Monitoren." Gruppe 2 besteht aus Standards unter dem allgemeinen Titel "Ergonomie der Mensch-Maschine Interaktion". --------- Iterativ ========= ========= Ein interativer Prozess wiederholt die Schritte des Prozesses des menschzentrierten Designs solange, bis eine Evaluation der Benutzungsschnittstelle zeigt, dass die Nutzungsanforderungen erfüllt sind. --------- Konsistenz ========= ========= Die gleiche Information wird in gleicher Weise innerhalb des gesamten interaktiven Systems entsprechend der Benutzererwartung dargestellt. Konsistenz ist eine Komponente der Erwartungskonformität. Bemerkung zum Seminar: Konsistenz ist auf verschiedenen Ebenen relevant wie zum Beispiel innerhalb einer Bildschirmanzeige, zwischen den Bildschirmanzeigen desselben interaktiven Systems, zwischen interaktiven System desselben Herstellers und zwischen ähnlichen interaktiven Systemen unterschiedlicher Hersteller. --------- Kontextuelles Interview ========= ========= Eine Technik zur Sammlung von Informationen über den Nutzungskontext und Erfordernisse. Der Interviewer (der User Requirements Engineer) stellt dem Benutzer Fragen über gegenwärtige Abläufe im Lichte des geplanten interaktiven Systems. Das Interview findet dort statt wo die Interaktion des Benutzers mit dem interaktiven System üblicherweise stattfindet, z.B. am Arbeitsplatz des Benutzers. Das kontextuelle Interview wird auf Englisch oft als "Contextual inquiry" bezeichnet. --------- Lernförderlichkeit ========= ========= Ein Dialogprinzip. Ein Dialog ist lernförderlich, wenn er den Benutzer beim Erlernen der Nutzung des interaktiven Systems unterstützt und anleitet. Empfehlungen zu Befolgung des Dialogprinzips:
  1. Der Dialog sollte ausreichende Rückmeldung über Zwischen- und Endergebnisse von Handlungen bereitstellen, damit die Benutzer von erfolgreich ausgeführten Handlungen lernen.
  2. Falls es zu den Aufgaben und den Lernzielen passt, sollte das interaktive System dem Benutzer erlauben, Dialogschritte ohne nachteilige Auswirkungen auszuprobieren.
--------- Low-Fidelity Prototyp ========= ========= Ein Prototyp, der sich auf die allgemeine Vorstellung und Struktur eines interaktiven Systems konzentriert. Er besteht oft aus Papierentwürfen und Bildschirmattrappen der Benutzungsschnittstelle eines zu entwickelnden interaktiven Systems. Ein solcher low-fidelity Prototyp wird üblicherweise von einem Menschen und nicht durch einen Computer betätigt. Er sollte so hinterlegt sein, dass er unmittelbar modifiziert werden kann. --------- Marktanforderung ========= ========= Eine Anforderung an ein interaktives System, die die Kaufentscheidung innerhalb einer anvisierten Benutzergruppe positiv beeinflusst. Soweit das interaktive System kostenlos ist, fördert die Marktanforderung die Entscheidung, das interaktive System zu benutzen. Beispiel: "Die Webseite muss mindestens so benutzbar sein wie jene der zwei stärksten Wettbewerber." --------- Meister-Schüler-Modell ========= ========= Ein Verhaltensprinzip für ein erfolgreiches Interview: Der Interviewer behandelt den Benutzer als den Meister während der Interviewer selbst der Schüler ist. Der Interviewer demonstriert nicht sein Wissen, sondern stellt seine Fragen mit dem Ziel etwas zu lernen. --------- Menschzentrierte Gestaltung ========= ========= Herangehensweise bei der Gestaltung und Entwicklung von interaktiven Systemen, die darauf abzielt, diese gebrauchtauglicher zu machen, indem sie sich auf die Verwendung des interaktiven Systems konzentriert und Kenntnisse und Techniken aus den Bereichen der Arbeitswissenschaft/Ergonomie und der Gebrauchstauglichkeit anwendet. --------- Mentales Modell ========= ========= Das Modell, das Menschen von sich, anderen, der Umgebung und den Dingen haben, mit denen sie interagieren. Menschen generieren mentale Modelle aufgrund von Erfahrung, Schulung und Anweisung. Das mentale Modell bezüglich eines interaktiven Systems wird größtenteils durch die Interpretation von wahrgenommenen Aktionen und sichtbaren Strukturen gebildet. Bemerkung zum Seminar: Alternative bekannte Definition: Der Denkprozess einer Person darüber, wie etwas in der realen Welt funktioniert. --------- Modell ========= ========= Im allgemeinsten Sinn ist ein Modell etwas, das irgendwie benutzt wird, um irgendetwas anderes zu repräsentieren. Zum Beispiel ist eine Zeichnung eines Zapfhahns ein Modell eines Zapfhahns. Eine Persona ist ein Modell eines Benutzers. --------- Moderator ========= ========= Eine Rolle in einem Usabilitytest oder in einer Fokusgruppe. Der Moderator leitet die Usabilitytestsitzung oder die Fokusgruppe. In einem Usabilitytest begrüßt der Moderator die Testteilnehmer, führt die Vorabinterviews durch, beobachtet die Testteilnehmer während der Erledigung der Aufgaben und macht das Interview nach dem Test. --------- Navigationsstruktur ========= ========= Die logische Organisation der angezeigten Informationseinheiten der Benutzungsschnittstelle. --------- Neutrale Frage ========= ========= Eine Frage in einem Interview, die keine impliziten Annahmen beinhaltet und auch keinen Ansatz bietet, einen Aspekt auszuschliessen oder die Antwort in eine bestimmte Richtung zu lenken. --------- Nutzungsanforderung - Qualitativ ========= ========= Eine Beschreibung, was Benutzer während der Durchführung einer Aufgabe mit dem interaktiven System erkennen, auswählen oder eingeben müssen. --------- Nutzungsanforderung ­ Quantitativ ========= ========= Geforderte Leistung, die die Basis für Design und die Evaluation eines interaktiven Systems darstellt, mit dem Ziel, identifizierte Erfordernisse zu befriedigen. --------- Nutzungsanforderung ========= ========= Was ein interaktives System aus Sicht des Benutzers ermöglichen soll. Nutzungsanforderungen können qualitativ oder quantitativ sein. --------- Nutzungskontext ========= ========= Benutzer, Aufgaben, Arbeitsmittel (Hardware, Software und Materialien) sowie die physikalische und soziale Umgebung in der das interaktive System genutzt wird. --------- Nutzungsobjekt ========= ========= Die wichtigsten Informationseinheiten auf oberster Ebene oder Daten mit denen Benutzer interagieren, um ihre Aufgaben durchzuführen. --------- Nutzungsproblem ========= ========= Ein Problem in der Benutzung der Benutzungsschnittstelle, das sich auf die Zufriedenstellung des Benutzers sowie die Effektivität und Effizienz des interaktiven Systems auswirkt. Nutzungsprobleme können zu Irritationen, Fehlern oder Verzögerungen führen oder sogar die Fertigstellung einer Aufgabe des Benutzers verhindern und machen die Benutzungsschnittstelle und damit das interaktive System weniger benutzbar für ihre Zielbenutzer. --------- Nutzungsszenario ========= ========= Eine narrative Beschreibung einer oder mehrerer Aufgaben und wie ihre Erledigung durch das interaktive System unterstützt wird. Das Nutzungsszenario beschreibt Schritt für Schritt das Vorgehen eines Benutzers, um die Aufgaben mit Hilfe des voraussichtlichen, sich noch in Entwicklung befindlichen interaktiven Systems zu erledigen. --------- Offene Frage ========= ========= Eine Frage in einem Interview, die keine besondere Erwartungen bezüglich Form oder Inhalt der Antwort beinhaltet. --------- Onlinehilfe ========= ========= Eine Form der Benutzerunterstützung. Onlinehilfe ist durch eine Computersoftware bereitgestellte themenorientierte oder prozedurale Information oder ein Verweis auf solche Information. Meist ist die Onlinehilfe dazu ausgelegt, Unterstützung für die Benutzung eines interaktiven Systems zu geben; sie kann aber auch Informationen zu weitergehenden Themenbereichen bieten. Wenn die Onlinehilfe unmittelbar auf die gegenwärtige Tätigkeit des Benutzers bezogen ist, dann spricht man von kontextsensitiver Hilfe. --------- Organisatorische Anforderung ========= ========= Anforderung an den Benutzer aus der Perspektive der Organisation. --------- Persona ========= ========= Eine beispielhafte Beschreibung eines Benutzers und was dieser bei der Benutzung eines interaktiven Systems tun möchte. Personas sind nicht Beschreibungen existierender Personen, sondern repräsentieren erfundene Beispiele eines realen Benutzers auf der Basis empirisch ermittelter Daten. Personas verfügen typischerweise über einen Namen, Alter, einige Hintergrundinformationen, Ziele und Wünsche. --------- Post-Test-Interview ========= ========= Eine Aktivität in einem Usabilitytest, bei der Testteilnehmer Fragen bezüglich ihres allgemeinen Eindrucks hinsichtlich der Gebrauchstauglichkeit des interaktiven Systems beantworten. Das Post-Test-Interview findet statt, nachdem der Testteilnehmer soviele der vorgehsehenen Aufgaben wie möglich in der vorgegeben Zeit erledigt hat. Manchmal als ,,Debriefing" bezeichnet. --------- Pre-Test-Interview ========= ========= Eine Tätigkeit bei einem Usabilitytest, bei der Testteilnehmer Fragen bezüglich ihres Hintergrundes und vorheriger Erfahrung mit dem interaktiven System oder ähnlichen interaktiven Systemen beantworten. Das Pre-Test-Interview findet statt, bevor der Testteilnehmer beginnt, die Testaufgaben abzuarbeiten. --------- Protokollant ========= ========= Eine Rolle in einem Usabilitytest. Der Protokollant ist ein Usabilityspezialist, der wichtigte Befunde während eines Usabilitytests notiert. Der Einsatz eines Protokollants erlaubt dem Moderator, sich voll auf den Testteilnehmer zu konzentrieren. --------- Prototyp ========= ========= Eine Repräsentation von Teilen oder des gesamten interaktiven Systems, die in einem bestimmten Maß für Analyse, Design und Evaluation benutzt werden kann. --------- Qualität ========= ========= Grad der Erfüllung von Anforderungen durch ein interaktives System. --------- Rekrutierungsfragebogen ========= Screener ========= Eine Reihe von Fragen für potenzielle Testteilnehmer hinsichtlich ihres Hintergrundes und Wissens. Ein Rekrutierungsfragebogen wird während der Rekrutierung von Testteilnehmern benutzt, um festzustellen ob Kandidaten über die erforderlichen Qualifikationen zur Teilnahme an dem Usabilitytest verfügen. --------- Rekrutierung (eines Teilnehmers für einen Usabilitytest) ========= ========= Anwerben von Benutzern, die die erforderliche Qualifikation haben, an einem Usabilitytest teilzunehmen. Oft wird ein Rekrutierungsfragebogen (Screener) eingesetzt, mit dem geprüft werden kann, ob Kandidaten über die erforderlichen Qualifikationen zur Teilnahme an einem Usbilitytest verfügen. --------- Ressourcen ========= ========= Alle nötigen Mittel zur Verwendung eines interaktiven Systems oder Dienstes. Typische Beispiele sind Zeit sowie motorische und mentale Anstrengung. --------- Review ========= ========= Eine von Laien, d.h. von Menschen mit begrenzter Usabilityerfahrung, durchgeführte Inspektion. --------- Rolle ========= ========= Ein Menge zusammenhängender Aspekte wie Verhalten, Rechte, Verpflichtungen und Normen in einer Arbeitssituation. Abhängig von der Komplexität eines Projektes können mehrere Menschen eine Rolle teilen oder mehrere Rollen einer Person zugewiesen werden. --------- Sekundäre Benutzer ========= ========= Menschen, die keine Benutzer sind, aber auf gewisse Weise vom interaktiven System betroffen oder beinflusst sind. Beispiele: Sekundäre Benutzer sind u.a. Führungskräfte von Benutzern, Vertriebspersonal und Vertreter, Projektmanager, Anwendungsentwickler, Trainer und das Kundenbetreuungsteam. Ist das interaktive System für die Benutzung zuhause gedacht, können sekundäre Benutzer auch Freunde oder Verwandte und Besucher des Benutzers sein. --------- Selbstbeschreibungsfähigkeit ========= ========= Ein Dialogprinzip. Ein Dialog ist selbstbeschreibungsfähig in dem Ausmaß, in dem es zu jeder Zeit dem Benutzer offensichtlich ist, in welchem Dialog er sich gerade wo befindet und welche Aktionen wie durchgeführt werden können. --------- Steuerbarkeit ========= ========= Ein Dialogprinzip. Ein Dialog ist steuerbar, wenn der Benutzer in der Lage ist, den Dialogablauf zu starten sowie seine Richtung und Geschwindigkeit zu beeinflussen, bis das Ziel erreicht ist. --------- Storyboard ========= ========= Eine Folge visueller Bildschirminhalte, die das Zusammenspiel zwischen einem Benutzer und einem vorgestellten interaktiven System darstellt. Ein Storyboard ähnelt oft einem Zeichtrickfilm. --------- Styleguide ========= ========= Eine Sammlung von Dialogprinzipen und Gestaltungsregeln die verwendet wird, um die grundsätzliche Erscheinung der Benutzungsschnittstelle auf jeglicher Software einer Organisation festzulegen. --------- Suche ========= ========= Eine Komponente der Benutzungsschnittstelle, die zum (Wieder)Auffinden von Nutzungsobjekten auf Basis fragmentaler Informationen zu diesem Objekt herangezogen wird. --------- Suggestionsfrage ========= ========= Eine Frage in einem Interview, die Annahmen oder einen Rahmen voraussetzt, der eine Präferenz für bestimmte Antwortmöglichkeiten vorgibt oder versucht, die Antwort in eine bestimmte Richtung zu lenken. --------- Summative Evaluation ========= ========= Eine Usability-Evaluation zur Feststellung, ob ein interaktives System die Nutzungsanforderungen erfüllt. Summative Evaluationen werden auch benutzt um den Grad der erreichten Gebrauchstauglichkeit eines interaktiven Systems zu bestimmten. Summative Evaluationen finden in der Regel dann statt, wenn ein interaktives System oder wesentliche Teile davon fertig entwickelt sind. --------- Testaufgabe ========= ========= Eine Beschreibung einer typischen Aufgabe, die ein Moderator einem Testteilnehmer während eines Usabilitytests stellt. --------- Testskript ========= ========= Eine von einem Moderator in einem Usabilitytest verwendete Checkliste, um den Überblick über die Fragen des Pre-Test-Interviews, die Testaufgaben und die Fragen des Post-Test-Interviews zu behalten. --------- Testteilnehmer ========= ========= Repräsentativer Benutzer, der typische Aufgaben in einem Usabilitytest löst. --------- Umgebung ========= ========= Die Gesamtheit der gegebenen Umstände oder Bedingungen, in denen ein Benutzer arbeitet. Diese umfassen die physische und die soziale Umgebung. --------- Usability ========= ========= Ausmaß, in dem ein interaktives System oder ein Dienst durch bestimmte Benutzer in einem bestimmten Nutzungskontext genutzt werden kann, um festgelegte Ziele effektiv, effizient und zufriedenstellend zu erreichen. --------- Usability Engineer ========= ========= Eine Rolle im Prozess der menschzentrierten Gestaltung. Der Usability Engineer betreut in einer Querschnittsfunktion verantwortlich den menschzentrierten Gestaltungsprozess. Er ist kompetent und erfahren in den Prozessschritten, verfügbaren Methoden der menschzentrierten Gestaltung und Gestaltungsregel für Benutzererlebnis, sodass er die Entwicklungsprojekte unter dem Aspekt menschzentrierter Gestaltung planen und die Qualität der Ausführung von menschzentrierter Gestaltungsbezogenen Aktivitäten sicherstellen kann. Dies beinhaltet die Integration der menschzentrierten Gestaltung in den Produktentwicklungsprozess des Unternehmens, die Definition von Erfolgskriterien für solche Projekte sowie das Training von beteiligten Projektteams. Er ist in der Lage, die für das jeweilige Projekt relevanten Arbeitsergebnisse, hierfür geeignete Vorgehensweisen sowie Arbeitshilfen und Festlegungen (z.B. Dialogprinzipien und Gestaltungsregel) für den menschzentrierten Gestaltungsprozess zu identifizieren und für ein Projekt festzulegen. --------- Usability Professional ========= ========= Eine Person, die qualifiziert und methodisch die Anforderungen an die Usability (Gebrauchstauglichkeit) interaktiver Systeme (Hardware und Software) herleitet, umsetzt oder deren Umsetzung überprüft. Der Usability Professional ist typischerweise in einem oder mehrerer der folgenden Arbeitsschwerpunkte spezialisiert:
  1. Analyse - Erheben von Nutzungskontexten, Herleiten von Nutzungsanforderungen
  2. Gestaltung - Konzipieren der Interaktion zwischen Mensch und interaktives System, strukturieren und darstellen handlungsleitender Informationen.
  3. Evaluation - Durchführen von inspektionbasierten Evaluationen (ohne Nutzer) und Usabilitytests (mit Nutzern)
  4. Prozessgestaltung und Methodeneinsatz - Festlegen, Einführen und Betreiben eines menschzentrierten Gestaltungsprozesses.
Die Grundlage der Tätigkeiten eines Usability Professionals sind die internationalen Normen zur Usability interaktiver Systeme und deren Gestaltungsprozess (ISO 9241) sowie aktuelles, publiziertes Fachwissen. Usability Professionals arbeiten in Projekten zu Herstellung von interaktiven Systemen und mit ihnen verbundener Dienstleistungen und Prozesse für den Einsatz in zahlreichen Projektkontexten (z.B. interaktive Systeme für den gewerblichen Einsatz, Haushaltsgeräte, interaktive Spiele, mobile Endgeräte). --------- Usability Tester ========= ========= Eine Rolle im Prozess der menschzentrierten Gestaltung. Der Usability Tester evaluiert Benutzungsschnittstellen in verschiedenen Stadien der Realisierung. Er definiert in Abstimmung der Prozessbeteiligten den Aufbau der Evaluationen und verantwortet die valide Ausführung der Evaluationen. Der Usability Tester erstellt entwicklungsbegleitende Usabilitytestberichte mit dem Ziel der Produktverbesserung sowie den Usabilitytestbericht über die abschließende summative Evaluation. Er kommuniziert diese Ergebnisse an die Interessenvertreter (Projektbeteiligten) oder betreut deren Kommunikation verantwortlich. --------- Usability-Evaluation ========= ========= Ein üblicher Ausdruck für Evaluation ­ inspektionsbasiert und Evaluation -­ benutzerbasiert. --------- Usabilitylabor ========= ========= Zwei oder mehrere Räume, die speziell für die Durchführung von Usabilitytests oder Fokusgruppen ausgestattet sind. Ein Usabilitylabor besteht of aus einem Testraum, wo die Testteilnehmer sitzen, und einem Beobachtungsraum, wo Interessenvertreter die Testteilnehmer beobachten können, während die Testteilnehmer Aufgaben lösen. Oft sind diese zwei Zimmer durch eine Spiegelwand getrennt, was den Beobachtern erlaubt, den Testteilnehmern zuzuschauen, nicht aber umgekehrt. --------- Usabilitytest ========= ========= Der Validierungsprozess zur Feststellung dass ein interaktives System alle vorher festgelegten Nutzungsanforderungen erfüllt. Ein Usabilitytest kann die Umsetzung der vorliegenden quantitativen und qualitativen Nutzungsanforderungen einschließlich Aufgabenvervollständigung, Aufgabendauer, Fehlerraten und Benutzerzufriedenstellung validieren. Ein Usabilitytest umfasst eine Reihe von Usabilitytestsitzungen. In jeder Sitzung versucht ein Testteilnehmer unter Benutzung des interaktiven Systems oder eines Prototypen eines interaktiven Systems repräsentative Aufgaben zu lösen. Üblicherweise werden Testsitzungen von einem Moderator moderiert und von einer Anzahl von Interessenvertretern beobachtet. Ein Protokollant zeichnet wichtige Befunde auf. Testen kann formal oder informal sein. Das Testen kann face-to-face (mit einem physisch anwesenden Moderator am gleichen Platz wie die Testteilnehmer) oder remote stattfinden und qualitative oder quantitative Daten ergeben. Das Testen kann zu jedem Zeitpunkt während der Entwicklungsphase geschehen ­ von der frühen Analysephase bis hin zur Lieferung des interaktiven Systems und darüberhinaus. Die Tests können sowohl an Papierentwürfen oder primitive Attrappen als auch an in Entwicklung befindlichen oder bereits fertiggestellten interaktiven Systemen durchgeführt werden. --------- Usabilitytest ­- Face-to-Face ========= ========= Ein Usabilitytest, bei dem sich die Testteilnehmer und der Moderator am selben physischen Ort befinden. --------- Usabilitytest ­- Remote ========= ========= Ein Usabilitytest bei dem sich die Testteilnehmer und der Moderator an unterschiedlichen Orten befinden. Der Moderator beobachtet die Testteilnehmer über eine Internetverbindung. Der Moderator kommuniziert mit den Testteilnehmern über Telefon oder Internetverbindung. --------- Usabilitytest­- Unbeaufsichtigt ========= ========= Ein Usabilitytest bei dem Testteilnehmer Testaufgaben lösen ohne dabei beobachtet zu werden. Handlungen werden auf Video zur späteren Analyse aufgezeichnet. --------- Usabilitytestbericht ========= ========= Ein Dokument, das die Ergebnisse eines Usabilitytests beschreibt. Ein Usabilitytestbericht beinhaltet eine Kurzdarstellung, 5-50 Befunde und das Testskript für den Usabilitytest. Oft umfasst der Usabilitytestbericht auch relevante Screenshots oder Bilder des getesteten interaktiven Systems. Manchmal auch als Testbericht bezeichnet. --------- Usabilitytestplan ========= ========= Eine kurze Beschreibung des Zwecks und Umfangs eines Usabilitytests. Der Plan beinhaltet die Anzahl geplanter Testteilnehmer, die ungefähre Länge jeder Testsitzung, den Namen des Moderators, einen Zeitplan und eine Kostenschätzung für den Usabilitytest einschließlich Personenstunden. --------- Usabilitytestsitzung ========= ========= Der Teil eines Usabilitytests, bei dem ein bestimmter Testteilnehmer repräsentative Aufgaben unter Benutzung des interaktiven Systems oder des Prototypen eines interaktiven Systems löst. --------- User Interface Designer ========= ========= Eine Rolle im Prozess des menschzentrierten Designs. Ein User Interface Designer implementiert das Benutzererlebnis basierend auf dem Design und den Nutzungsszenarios, des Interaktionsdesigners. Der User Interface Designer erzeugt interaktive Prototypen. --------- User Requirements Engineer ========= ========= Eine Rolle im Prozess des menschzentrierten Designs. Der User Requirements Engineer identifiziert und beschreibt den tatsächlichen Nutzungskontext von Benutzern. Daraus leitet der User Requirements Engineer die Nutzungsanforderungen und damit verbundene Organisatorische Anforderungen ab, die für ein bestimmtes Projekt umgesetzt werden müssen. Der User Requirements Engineer priorisiert die Anforderungen um sicherzustellen, dass die grundsätzlichen Erfordernisse des Benutzers erfüllt werden. --------- Wireframe ========= ========= Grobe Skizze von Navigations- und Inhaltselementen, die eine Benutzungsschnittstelle ausmachen. Typischerweise werden dabei das visuelle Design und das exakte Layout ignoriert. Auf Deutsch auch ,,Drahtgeflecht" genannt. --------- Zufriedenstellung ========= ========= Freiheit von Beeinträchtigungen und positive Einstellungen gegenüber der Nutzung des interaktiven Systems.
